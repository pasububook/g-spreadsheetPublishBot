# SUGGEST_DB
## 1. 概要
`SUGGEST_DB` は、条件リストエディタにおいてユーザーがタグ（<, ＜, `）またはメンショントリガー（@）を入力した際に表示される「入力候補（サジェスト）」のデータソースを管理する定数オブジェクト（配列）です。現在は `src/config.gs` に定義されており、Viteのビルド機構（`config.js`）を通じてフロントエンドに配信されます。

このデータベースは、単なる名称のリストではなく、**多角的な検索（ローマ字、ひらがな、略称）** を実現するための検索インデックス、および**教科情報の自動入力**のための参照データとしての役割も担っています。


### データベースの設計方針・特徴

本データベースは、ユーザーのスムーズな入力を支援するため、以下の要件を満たすように設計およびデータ登録がなされています。

- **網羅性と正確性**  
  シラバスと購入リスト（画像等）を厳密に照合し、すべての科目の教材を妥協なく「正式名称」で登録。
- **入力揺れへの完全対応**  
  タイポ、短縮名、略称、さらには英語・ローマ字の入り混じり（例: `sksheed` 等）など、様々な表記揺れを吸収。
- **誤入力（類似音）への対応**  
  発音やスペルの似ている音（例: `read` と `lead` 等）の誤入力ケースでも、正しく対象がヒットするようにキーワードを整備。

## 2. データ構造定義
`SUGGEST_DB` は、以下のプロパティを持つオブジェクトの配列として定義されます。

```js
const SUGGEST_DB = [  
  {  
    name: String,      // [必須] タグとして表示される正式名称  
    type: String,      // [必須] カテゴリ分類（タグの色分けに使用）  
    subject: String,   // [任意] 教科名（教科プルダウンの自動選択に使用）  
    keywords: Array    // [必須] 検索用キーワードの配列  
  },  
  // ...  
];
```

### **2.1 プロパティ詳細**
| プロパティ名 | 型 | 説明 | 制約・備考 |
| :---- | :---- | :---- | :---- |
| **name** | String | サジェストリストおよび確定後のタグに表示される文字列。 | 一意（Unique）であることが望ましい。IDとして機能する。 |
| **type** | String | タグの背景色・アクセントカラーを決定するための分類キー。 | TAG_CATEGORIES 定数のキーと一致させる必要がある。 一致しない場合はデフォルト（その他）のデザインが適用される。 |
| **subject** | String | 教材に関連する教科名。 タグ挿入時に、行の教科選択プルダウンを自動設定するために使用される。 | SUBJECT_LIST 定数に含まれる値（"国語", "数学" 等）を指定する。 該当なしの場合は省略可（undefined）または null。 |
| **keywords** | Array<String> | ユーザーの入力に対してヒットさせるための検索語句リスト。 | 全て小文字（lowercase）で登録することを推奨（検索ロジックで正規化されるため）。 |

## **3. カテゴリ定義 (TAG_CATEGORIES) との連携**

type プロパティは、以下の定義済みカテゴリと紐付き、サジェスト表示時およびタグ挿入時に色が反映されます。

- **教科書** (#D6EAF8 / 青系)  
- **資料集** (#D5F5E3 / 緑系)  
- **ワーク** (#FAE5D3 / オレンジ系)  
- **プリント** (#E8DAEF / 紫系)  
- **その他** (#F2F4F4 / グレー系)

**Note:** `SUGGEST_DB` に登録のない type を指定した場合、または未登録の単語が入力された場合は、自動的に「その他」の配色が適用されます。

## **4. 教科定義 (SUBJECT_LIST) との連携**
subject プロパティは、以下の教科リストと対応します。
- 国語  
- 数学  
- 英語  
- 理科  
- 社会  
- その他

## 5. 検索キーワード (keywords) の登録運用ルール
検索ヒット率（UX）を最大化するため、keywords 配列には以下のパターンの登録を推奨します。

### 5.1 必須パターン

1. **ローマ字 (ヘボン式/訓令式)**  
   - 例: "しゃかい" -> "shakai", "syakai"  
2. **ひらがな**  
   - 例: "社会" -> "しゃかい"  
3. **頭文字 (Acronym)**  
   - 例: "近現代文学" -> "k", "kb"

### 5.2 推奨パターン（揺らぎ対応）

- **英語表記が含まれる場合**: その単語の読みとスペル。  
  - 例: "サイエンスガイドZ" -> "science", "guide", "z", "さいえんす"  
- **複合語の分割登録**: ユーザーが単語の途中から入力し始めた場合に対応するため。  
  - 例: "現代社会と情報" -> "gendai", "shakai", "joho"

## 6. データサンプル

```js
const SUGGEST_DB = [  
  {   
    name: "現代社会と情報",   
    type: "教科書",

      "社会", "しゃかい", "じょうほう", // 日本語・ひらがな

    ]   
  },

    type: "教科書",   
    subject: "国語",  
    keywords: [  
      "dokkai", "d",                // 略称・頭文字  
      "kyoka", "k",   
      "tekisuto", "t",  
      "どっかい", "きょうか", "てきすと" // ひらがな  
    ]   
  },

```

## 7. タグのシリアライズ仕様（保存フォーマット）
### 7.1 保存フォーマット
タグはスプレッドシートへの保存時、以下の形式の文字列に変換されます。

```
<name;type>
```

| 要素 | 説明 |
| :---- | :---- |
| **name** | タグ名称（サニタイズ済み）。セミコロン `;` は含まれない。 |
| **type** | タグのカテゴリ（TAG_CATEGORIES のキー）。未設定時は `その他` が使用される。 |

### 7.2 保存フォーマット例
| タグ表示名 | タイプ | 保存後の文字列 |
| :---- | :---- | :---- |
| 読解力強化テキスト | 教科書 | `<読解力強化テキスト;教科書>` |
| サイエンスガイドZ | 資料集 | `<サイエンスガイドZ;資料集>` |
| 課題プリント | その他 | `<課題プリント;その他>` |

## 8. タグ内 type 指定構文
### 8.1 構文定義

タグ入力時（トリガー文字 `<`, `＜`, `` ` `` の後）に、以下の形式で **type を直接指定** できます。

```
<name;type>
```

- **name**: タグ名称。  
- **type**: TAG_CATEGORIES のキー（"教科書", "資料集", "ワーク", "プリント", "その他"）。  
- **区切り文字**: `;`（半角セミコロン）。

### **8.2 動作仕様**

| 条件 | 動作 |
| :---- | :---- |
| `;` あり かつ type が TAG_CATEGORIES に存在する | 指定された type を使用してタグを生成。 |
| `;` あり かつ type が TAG_CATEGORIES に存在しない | `その他` にフォールバック。`SUGGEST_DB` のヒットがあればそちらを優先。 |
| `;` なし | 従来通り `SUGGEST_DB` を参照して type を決定。 |

**入力例**

```
<課題プリント;プリント>  →  type="プリント" のタグが生成される
<未登録教材;ワーク>      →  type="ワーク" のタグが生成される（新規作成）
<読解力強化テキスト>     →  SUGGEST_DB から type="教科書" が自動取得される（従来動作）
```

### 8.3 サジェスト表示時のプレビュー

クエリに `;` が含まれる場合、サジェストボックスの「新規作成」候補に指定された type のカラーバッジが表示されます。これによりタグ確定前に適用 type を視覚的に確認できます。

---

## 9. name のサニタイズ仕様
タグ名称（name）に `;`（半角セミコロン）が含まれる場合、保存フォーマットの区切り文字と衝突するため、自動的に `；`（全角セミコロン U+FF1B）へ置換されます。

| 入力 | 保存後 |
| :---- | :---- |
| `A;B` | `A；B` |
| `abc;def;ghi` | `abc；def；ghi` |

この処理は `insertTag()` 関数内で行われ、サジェストからの確定・自動確定（`>` 入力）・インラインサジェスト確定のすべての経路に適用されます。

## 10. 検索・表示ロジックの仕様（参考）

現在の実装 (handleEditorInput および showSuggestions) では、トリガー文字によって挙動が異なります。

### 10.1 トリガー文字

- タグ作成モード: `<`, `＜`, `` ` ``  
- メンション/参照モード: `@`

### 10.2 フィルタリング仕様

1. **正規化**: ユーザーの入力値を小文字化 (toLowerCase())。  
2. **Name一致**: name 自体に部分一致するか確認。  
3. **Keyword前方一致**: keywords 配列内のいずれかの文字列が、入力値で**始まっているか (startsWith)** を確認。

### 10.3 モードによる挙動の違い

| 機能 | タグ作成モード (< 等) | メンションモード (@) |
| :---- | :---- | :---- |
| **未登録単語の表示** | 検索にヒットしない場合、「新規作成」候補として表示する。 | **表示しない**（登録済みの単語のみ候補に出す）。 |
| **自動確定** | `>` 等の閉じ文字入力で、選択中の候補または入力値をタグ化する。 | **無効**（閉じ文字による自動タグ化は行わない）。 |
| **type プレビュー** | クエリが `name;type` 形式の場合、新規作成候補に指定 type のバッジを表示。 | **無効** |

```js
// 検索ロジック抜粋（v2.5.0 以降）  
if (query !== "" && suggestionTrigger !== "@") {  
  const exists = filteredSuggestions.some(i => i.name === query);  
  if (!exists) {  
    // name;type 形式からプレビュー type を抽出  
    let previewType = "その他";  
    const sepIdx = query.indexOf(';');  
    if (sepIdx !== -1) {  
      const parsedType = query.substring(sepIdx + 1).trim();  
      if (parsedType && TAG_CATEGORIES[parsedType]) previewType = parsedType;  
    }  
    filteredSuggestions.push({ name: query, type: previewType, isNew: true });  
  }  
}  
```
