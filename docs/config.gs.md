# `src/config.gs` ドキュメント

`src/config.gs` は、本プロジェクト全体にわたる各種設定値を一元管理する設定ファイルです。Google Apps Script（GAS バックエンド）およびサイドバーなどのウェブフロントエンド（UI）の両方から参照される設計となっており、変更管理から自動通知までの動作を一括して制御します。

## 定義されている主な設定値

### 1. ドキュメントの基本設定
- **`docTitle`**: メインシート（ドキュメント本体）のタイトル。（例: `"dev_2年次_秋季休業課題・テスト"`）
- **`isShowCheckbox`**: チェックボックス列を表示するかの `true/false`。
- **`isProtectEditSheet`**: 編集用のシートの保護を有効にするかの `true/false`。

### 2. 教科データ (`subjectData`)
- **概要**: 大分類（「国語」「数学」など）と、それに紐付く科目一覧の階層・カテゴリ構造を定義するオブジェクトです。
- **関連ファイル**: `src/auto_setup.gs` などで、シートの初回セットアップ時に教科ごとの構成（行の作成等）の基準として利用されます。

### 3. 教科正式名称リスト (`SUBJECT_LIST`)
- **概要**: 利用可能な教科の「正式名称」を管理する配列です。（例: `"国語/現代文α"`）
- **関連ファイル**: フロントエンド（サイドバーなど：`src/html-src/scripts/changesInput/*`）でのバリデーションや入力画面のセレクトボックスの構築に利用されます。

### 4. サジェストデータベース (`SUGGEST_DB`)
- **概要**: 入力画面でユーザーがタグやメンショントリガー（< など）を入力した際に表示される入力候補（サジェストデータ）の一覧です。タイプミスの揺れや短縮名に完全対応するための `keywords` を含みます。
- **関連ファイル**: フロントエンド用スクリプト（`src/html-src/scripts/changesInput/config.js` など）にロードされサジェスト機能に利用されます。詳細な仕様と役割については [`SUGGEST_DB.md`](SUGGEST_DB.md) を参照してください。

### 5. タグカラー定義 (`TAG_COLORS`)
- **概要**: 教材のタイプ（「教科書」「資料集」「ワーク」など）それぞれに対するカラーコード（HEX値）を定義します。
- **関連ファイル**: `src/notice.gs` 等から参照され、Google Chatなどのチャットツールへの連携・通知時にメッセージの見栄え（カードカラー）を装飾するために利用されます。

## 他ファイルとのシステム的な連携（データフロー）

本プロジェクト内での `config.gs` の最大の特徴は、**「GAS（バックエンド）とHTML（フロントエンド）の設定の単一情報源（Single Source of Truth）」** として機能している点です。

### 1. Google Apps Script 側での挙動
GAS における `.gs` ファイル群はグローバルにロードされるため、`src/config.gs` で宣言された定数はそのまま `auto_setup.gs` や `notice.gs` から直接呼び出されます。

### 2. フロントエンド側への連携機構（Viteビルド）
バックエンドである `config.gs` の内容をフロントエンド（HTML/JS側）でも共有するため、ビルドスクリプトによる連携が構築されています。

1. **`build.js` による注入**: \n   `npm run build` を実行時、Vite は [`build.js`](../build.js) 内の `define` オプションによって、`config.gs` のテキスト文字列全体を `__CONFIG_GS_RAW__` という変数として埋め込みます。
2. **フロントでのパース**: \n   フロントエンド用スクリプトの [`src/html-src/scripts/changesInput/config.js`](../src/html-src/scripts/changesInput/config.js) はこの定数を読み込み、関数化して評価することで `SUBJECT_LIST` や `SUGGEST_DB` などの設定値を抽出します。

これにより、プロジェクト内のどの環境からも同じ `config.gs` の設定を参照可能にし、二重管理を防いでいます。
